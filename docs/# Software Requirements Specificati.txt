# 1. Introduction

## 1.1 Purpose

This document defines the software requirements for **Cogna**, an API-first AI subscription marketplace that enables customers and businesses to purchase, manage, and resell premium AI subscriptions through a secure web platform and REST API.

## 1.2 Scope

Cogna provides a centralized platform for selling digital AI subscriptions such as:

- ChatGPT Plus
- Gemini Pro
- Claude Pro
- Cursor Pro
- Perplexity Pro
- GitHub Copilot
- Midjourney
- Additional AI products

The platform integrates with external providers (initially the Akunding Store Reseller API) to automate subscription fulfillment while exposing its own API for third-party developers.

---

# 2. Product Overview

Cogna consists of four main applications:

- Public Website
- Customer Dashboard
- Admin Dashboard
- Developer Portal

All applications communicate with a centralized backend through a REST API.

# 8. Third-Party Integrations

## Payment Gateways (Multi-Gateway)

- Paystack (primary)
- Monnify (secondary)
- Architecture supports adding additional gateways without breaking changes
- Each product can be assigned a specific payment gateway
- Gateway config stored per-product and in platform settings

## AI Subscription Providers (Per-Product)

- Akunding Store Reseller API (initial provider)
- Each product stores its own provider_id and provider-specific API config
- New providers can be added without modifying core order logic
- Provider credentials are stored per-provider, not globally

# 15. Success Criteria

The MVP will be considered successful when it can:

- Register and authenticate users.
- Display products from the provider.
- Process and verify payments.
- Create and manage orders.
- Track order status.
- Deliver completed subscription details.
- Provide an admin dashboard.
- Provide a developer API with API key authentication.
- Maintain secure and reliable operations.

---

# Project Summary

Cogna is an API-first AI subscription marketplace designed to simplify the purchase and distribution of premium AI subscriptions. The platform includes a public storefront, customer dashboard, developer portal, and administrator dashboard, all powered by a centralized REST API. It integrates with external reseller providers and payment gateways to automate fulfillment while offering secure APIs for third-party developers. The architecture is designed to be scalable, secure, and extensible, enabling future support for multiple providers, white-label integrations, and additional digital products.
