# Coding Standards & Development Guidelines

## Cogna Platform

Version: 1.0

---

# General Principles

- Keep code simple and maintainable.
- Follow clean architecture principles.
- Separate business logic from controllers.
- Never duplicate business logic.
- Write reusable components and services.
- Keep functions focused on a single responsibility.

# Validation

Use request validation for every endpoint.

Never trust client input.

Return descriptive validation errors.

---

# Database Rules

- UUID primary keys
- Foreign keys for relationships
- Use transactions for payments and orders
- Soft delete where appropriate
- Never delete financial records

---

# Logging

Log

- Login attempts
- Payments
- Orders
- Provider requests
- Errors
- Admin actions

---

# Error Handling

Use centralized error handling.

Never expose internal server errors.

Return consistent JSON responses.

# Security

- HTTPS only
- Hash passwords with bcrypt
- Store secrets in environment variables
- Enable rate limiting
- Validate all inputs
- Sanitize user data
- Verify webhook signatures
- Never expose provider credentials

# Development Workflow

1. Design database changes.

2. Update Prisma schema.

3. Generate migrations.

4. Implement services.

5. Implement controllers.

6. Add validation.

7. Write tests.

8. Update documentation.

9. Deploy to staging.

10. Deploy to production.

---

# Code Quality

- Use TypeScript strict mode.
- Avoid duplicated code.
- Keep controllers thin.
- Keep business logic inside services.
- Use reusable utility functions.
- Write descriptive variable and function names.
- Document complex business logic.

---

# Project Goal

Build a scalable, secure, API-first AI subscription marketplace that is easy to maintain, easy to extend, and ready for future integrations with multiple providers, payment gateways, and third-party applications.
